anisble, roots, and a bit of history

  • Ansible was created by Michael DeHaan, the author of the provisioning server application Cobbler (REDHAT 2008 ish?) and co-author of the Func framework for remote administration.
  • Ansible is 4 years old, the first release was February 20, 2012.
  • It's github growth stats are remarkable
  • On 16 October, 2015, Red Hat Inc. announced that it would acquire Ansible, Inc.

marketing hype: It's automation for everyone!

Deploy apps, Manage systems, Crush complexity, build a strong foundation for DevOps, continuous delivery and document your infrastructure with readable recipes...

Real world usage:

  • Provisioning
  • Configuration Management
  • Application Deployment
  • Continuous Delivery
  • Security and Compliance
  • Orchestration
  • Specific or immediate task execution

a splash of ansible trivia

In case you where wondering, the name "Ansible" references the fictional instantaneous hyperspace communication system featured in Orson Scott Card's Ender's Game, and originally invented by Ursula K. Le Guin for her 1966 novel Rocannon's World).

playbooks, roles, tasks, handlers and modules

Ansible's buidling blocks

Playbooks: organize roles, tasks and handlers
Roles: start with playbooks
Tasks: Copy a file, install some things, get a value
handlers: Special tasks, usually used for restarting services
(sets a flag to restart a service at the end of a playbook)
modules: ansible comes with many, many modules. Switch configuration to clouds and clusters.
